# Multi-Stage Docker Build

**Path:** `multi-stage docker build`  
**Kind:** `concept`

> A Dockerfile pattern using multiple FROM instructions to separate build-time and runtime concerns. Intermediate stages are discarded, producing a minimal final image containing only what is needed to run the application.
